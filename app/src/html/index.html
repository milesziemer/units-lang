units interpreter

units interpreter allows you to perform calculations in a familiar and intuitive way, without having to worry about converting between different si units

use the input prompt below just as you would any calculator. units interpreter has all the standard operations, follows order of operations, parenthesis etc. you can see your calculations tracked above the input
when a value is going to be reused, it is best to store that value in a variable rather than typing it out each time. units interpreter allows you to do this with variable assignments:
units > let x = 25 * 3 / (1 + 5)
 x is now assigned to the value of the arithmetic on the right side of the = and can be used later like any other value.
in order to perform calculations with units, they must be specified or annotated. this can be done multiple ways:
  • in variable assignments with :
    units > let x: km = 3.2
  • next to the number
    units > 15 in + 3.2 cm
  • as an operation with :
    units > (1 in + 2 cm): ft
more complex operations are supported via functions. units interpreter currently has a few built-in functions:
  • basic trig sin cos tan
  • natural logarithm ln
functions can be called by passing a variable or number within parenthesis like so:
units > sin(3.14)
 all current functions have a unitless output.
only units that are supported by units interpreter can be used. others will cause an error. currently the only supported units are basic length/distance:
  • imperial in ft yd mi
  • metric mm cm m km
all length/distance units can be converted to and from one another.
currently there are no built-in constants, but there are plans to add support for pi and e
